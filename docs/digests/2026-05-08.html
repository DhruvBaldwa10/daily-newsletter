Anthropic's Big Bet, Claude's Holes

Anthropic locked in a $5B/yr compute deal with SpaceX's Colossus while simultaneously shipping security vulnerabilities in Claude Code. OpenAI flooded the zone with voice models, ads, and GPT-5.5. A lot happened — here's what it means for builders.

Anthropic x SpaceX: The $5B Compute Play Nobody Saw Coming

The biggest story out of Anthropic's Code w/ Claude 2026 event wasn't a model launch — it was a deal. Anthropic announced it has secured all of the capacity of SpaceX's Colossus data center, a 300MW facility, in what amounts to a ~$5B/year compute commitment. Latent Space reports the company's ARR growth is now running at 8,000% annualized — a number that would be unbelievable if the compute contracts weren't providing independent validation.

Simon Willison's notes on the deal point out the obvious elephant in the room: Colossus is the Memphis facility with the controversial environmental record around gas turbines and air pollution. Anthropic's safety-first brand is now entangled with some genuinely thorny optics. But strategically, this is Anthropic signaling it can compete on compute scale — and that it's willing to partner with whoever can deliver the GPUs.

For builders, the immediate read-through is on usage limits. Higher limits for Claude are being rolled out as a direct result of this capacity expansion. If your product has been throttled by Claude rate limits — especially in agentic, long-running workflows — this is your cue to re-evaluate what's now possible at scale.

Claude Code Has a Security Problem — Two, Actually

The same week Anthropic is celebrating capacity growth, Claude Code is shipping with serious vulnerabilities. Two separate disclosures landed this week. First, The Register reported a one-click RCE via Claude Code's trust prompt mechanism — and Anthropic's official response was essentially "you shouldn't have clicked OK." The security community has been less than impressed with that framing. Second, a CVE filed as CVE-2026-39861 documents a sandbox escape via symlink traversal in Claude Code itself.

These aren't hypothetical academic vulnerabilities. If you're deploying Claude Code in team environments, CI pipelines, or giving it access to production credentials — both of these are live threat vectors right now. The symlink escape is particularly nasty because it undermines the core security promise of sandboxed code execution, which is the whole premise of agentic coding tools operating safely on real repos.

The timing is notable: Armorer and Agentctl both launched on HN this week as "local control planes for AI agents" — the market is clearly sensing a gap in agent security primitives. Builders shipping agentic coding workflows need to treat agent sandboxing as a first-class infrastructure concern, not an afterthought. The convenience of one-click approval flows is also a one-click attack surface.

Claude Mythos Hardens Firefox — A Glimpse at Security-Specialized AI

Buried in the week's noise is one of the most technically impressive AI-assisted security stories yet. Simon Willison flagged Mozilla's detailed writeup of how they used Claude Mythos Preview — Anthropic's unreleased research model — to find and fix hundreds of vulnerabilities in the Firefox codebase. The Mozilla team's framing is striking: they describe a phase transition where AI-generated bug reports went from "mostly noise" to "suddenly very good."

This is the clearest production signal yet that AI-assisted security research has crossed from parlor trick to genuine force multiplier. The key unlock appears to be Mythos's ability to reason about complex control flow across large codebases — the kind of multi-file, multi-hop reasoning that prior models consistently failed at. For builders maintaining security-sensitive codebases, this suggests a near-term future where AI-assisted fuzzing and vulnerability scanning becomes table stakes infrastructure, not a research experiment.

It also reframes the Claude Code security vulnerabilities discussed above: the irony of Anthropic's model hardening Firefox while Claude Code ships sandbox escapes is thick. The lesson for product builders is that AI security capability is deeply use-case specific — the same underlying model can be exceptional at finding vulnerabilities in one context and introducing them in another.

"Suddenly, the bugs are very good. Just a few months ago, AI-generated security bug reports to open source projects were mostly known as noise."

— Mozilla Hacks, via Simon Willison

Vibe Coding and Agentic Engineering Are Converging — Uncomfortably

Simon Willison published a thoughtful reflection this week titled "Vibe coding and agentic engineering are getting closer than I'd like." The core tension: vibe coding (iterate fast, don't fully understand the output) and agentic engineering (AI autonomously executes long multi-step tasks) are conceptually opposite safety postures — but in practice they're blending in his own workflow. When you're reviewing diffs from an agent that wrote code you didn't spec in detail, you're vibe-reviewing agentic output. That's a new and underexplored failure mode.

Eugene Yan's recent post on working with AI provides a useful counterframe: "context as infrastructure, taste as config, verification for autonomy, scale via delegation." The argument is that builders who invest in verifiable outputs — good evals, clear contracts between agent steps — can safely push autonomy higher. The builders who skip that infrastructure are the ones who get burned.

The "IDE as Operating System for AI" essay circulating on HN gestures at where this goes architecturally: the IDE needs to become an environment that manages agent processes, their permissions, their outputs, and their sandboxes — not just a text editor with autocomplete. The Claude Code CVEs this week are arguably the opening act of that reckoning.

"Context as infra, taste as config, verification for autonomy, scale via delegation, closing the loop."

— Eugene Yan, "How to Work and Compound with AI"
